# コンセプト

本ドキュメントは jcd-editor のプロダクト思想を記録するものです。意図的に短く保ちます。将来の機能がここに記された原則と矛盾する場合、明示的に改訂されるまでは原則が優先されます。

## このツールが何であるか

ユーザーが自分のマシンで所有・保持する構造化されたキャリア情報から、日本式の履歴書および職務経歴書を生成する、local-first なエディタです。

## 原則

### 1. キャリア情報はユーザー本人の手元に置く

ユーザーのキャリア情報は本人のものです。本ツールは、入力・編集・プレビュー・エクスポートというワークフロー全体を、外部サービスへデータを送らずに完結できるよう設計されます。

### 2. 広告収益型の前提を採用しない

キャリア文書は高度にセンシティブな個人情報 (職歴・希望年収・連絡先・属性情報) を含みます。本ツールのアーキテクチャは、広告型あるいはデータ収集型のビジネスモデルを前提としません。将来のクラウド版あるいは SaaS 派生があるとしても、それは opt-in であり、ローカル利用の前提条件にしてはなりません。

### 3. 完全ローカルでの利用を保証する基準とする

インターネット接続もサードパーティーアカウントも持たないユーザーであっても、以下を行えなければなりません。

- キャリア情報の入力と編集
- レンダリングされた文書のプレビュー
- PDF / JSON / Markdown / HTML へのエクスポート
- セッションをまたいでデータを保存・再読み込み

将来クラウド同期を追加する場合も、この基準を壊してはなりません。

### 4. AI は補助機能であり、中核ではない

本ツールの中核価値は、構造化された執筆と信頼できるレンダリングであり、AI による生成ではありません。AI 補助は導入時においても optional なアダプタであり、無効化・差し替え・ローカルモデルへの切り替えができなければなりません。

### 5. 外部 AI への送信は明示的な確認を必要とする

キャリア情報を外部の AI プロバイダーへ送信する場合は、以下を満たさなければなりません。

- デフォルト無効
- 送信先 (ローカル vs 外部) および送信内容を明示する
- 既定では、ユーザーが選択した範囲のみを送る (全文ではない)
- 全文送信には明示的な確認ステップを必須とする

### 6. スコープ: 日本式の書類

初期スコープは日本式の書類、すなわち履歴書 (JIS 様式) と職務経歴書です。Western CV / résumé の対応は将来検討事項であり、日本式フォーマットのデータモデルを薄めることを許容してはなりません。
