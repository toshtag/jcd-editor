# ADR 0001: Local-first を中核方針とする

## Status

Accepted (2026-05-19)

## Context

jcd-editor は履歴書 (履歴書) および職務経歴書を生成・編集するためのツールであり、扱うデータは強くセンシティブな個人情報を含みます。具体的には以下のとおりです。

- 氏名、生年月日、住所、連絡先などの直接的な個人識別情報
- 学歴・職歴・資格などの経歴情報
- 希望条件 (希望年収、希望勤務地など) を含む転職活動に関わる情報
- 場合により、推薦状や評価情報など第三者由来のセンシティブ情報

このカテゴリーのデータを SaaS のみで提供すると、以下の構造的リスクをユーザーに負わせることになります。

- **ベンダーロックイン**: データの可搬性が損なわれ、ユーザーが将来のサービス変更を選びにくくなる
- **シャットダウンリスク**: 事業継続性がプロダクトを使う前提条件になる
- **データ収集・広告連動の懸念**: 「無料 SaaS」は構造的に広告・データ収集を選好しやすく、扱うデータの性質と相容れにくい
- **ネットワーク・アカウント必須化**: 利用に毎回オンライン・第三者アカウントが要求される

一方、ローカルファースト・アプローチでは:

- データはユーザー自身のマシンに留まる
- オフラインでも完結する
- データの可搬性 (JSON エクスポート等) が容易
- 将来クラウド同期を「追加」できる選択肢を残せる

代償としては:

- 自動的なクラウド同期がなく、複数デバイスでの即時共有は別途実装が必要になる
- バックアップは利用者の責任が大きくなる
- 配布・自動更新の仕組みが SaaS より複雑になる場合がある

## Decision

jcd-editor は **local-first を中核方針とする** ことを決定します。

具体的には以下を満たす設計を取ります。

1. 入力・編集・プレビュー・エクスポートを含む主要ワークフローを、外部サービスへ送信せずに完結可能にする
2. データはユーザーが指定したローカルディレクトリ (ファイルシステム) に保存することを既定とする
3. クラウド同期 / SaaS 派生は完全に optional とし、ローカル利用の前提条件にしない
4. 外部 AI を含む任意のデータ送信は、デフォルト無効・明示確認の対象とする
5. 広告型・データ収集型のビジネスモデルを前提としない

## Consequences

**良い結果:**

- ユーザーが自分のデータを完全に管理できる
- ネットワークやサードパーティーアカウントへの依存を最小化できる
- データの可搬性が確保され、別ツールへの移行が容易になる
- AI 利用や外部サービス利用の判断をユーザー自身に委ねられる

**負担となる結果:**

- 自動同期や複数デバイス共有が「追加機能」として後発になる
- バックアップとデータ消失の責任がユーザー側に寄る
- 配布・アップデート・テンプレート共有の仕組みを別途整える必要がある

**これにより制約される後続判断:**

- Storage はローカルファイル (まず JSON、後に SQLite 等) を中心に設計する
- PDF 生成は Playwright 等を含めローカル実行を前提に設計する
- AI 連携を入れる場合も、ローカル LLM (Ollama 等) を一級市民とする
