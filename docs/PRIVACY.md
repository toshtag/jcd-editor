# プライバシーポリシー (リポジトリ運用)

本ドキュメントは、jcd-editor リポジトリで個人情報を扱う際の運用規則を定めます。エンドユーザー向けのプライバシーポリシーではなく、リポジトリのコントリビューターが守るべきルールです。

## 基本原則

実在する個人の履歴書・職務経歴書データを、いかなる形でも本リポジトリにコミットしてはなりません。

## 禁止事項

以下はコミット対象外です。

- 実在する個人の氏名、フリガナ、生年月日、住所、電話番号、メールアドレス
- 実在する企業名・組織名・部署名 (応募先・前職等を問わず)
- 実年収、希望年収、賞与など、金額の具体値
- 実在する案件・プロジェクトの社外秘情報
- 面接ログ、面接対策メモ、自己分析シート、推薦状
- 認証情報 (API キー、OAuth トークン、パスワード)
- AI へのプロンプトおよび応答ログ (履歴書本文以上にセンシティブになる場合があります)
- 生成された PDF / DOCX / XLSX (個人情報を含む可能性が高いため)

## サンプルデータ規約

サンプルやテスト用フィクスチャに含めてよいのは、ダミーデータのみです。

- 氏名: 「山田 太郎」「鈴木 花子」のような明確な仮名
- 企業名: 「株式会社サンプル」「example.com」のような明確な仮想組織
- メールアドレス: `dummy@example.com` 等、RFC 2606 で予約された example ドメイン
- 電話番号: 03-0000-0000 等、明らかにダミーと分かる番号
- 写真: 実在の人物が判別できない汎用イラストまたは無地のプレースホルダー

実在人物・実在組織を想起させる「もっともらしいダミー」も避けてください。

## ディレクトリの扱い

`.gitignore` で除外する個人データ領域:

| パス | 用途 |
| --- | --- |
| `private/` | コントリビューター個人の作業メモ |
| `.local/` | ローカル設定 |
| `data.local/` | 実データの一時保管 |
| `outputs.local/` | 生成物の個人保管 |
| `outputs/` | 生成された PDF / HTML 等 (個人情報を含むため git 管理しない) |
| `ai-logs/` | AI とのやり取りログ |
| `prompts.local/` | AI プロンプトの草稿 |
| `*.local.json` / `*.local.md` / `*.local.pdf` | 任意ファイルの個人扱い指定 |
| `.env` / `.env.*` | シークレット |

生成物の保管場所:

- 個人用生成物は `outputs/` または `outputs.local/` に置きます (どちらも git 管理外)
- 検証目的のサンプル成果物 (完全に無害化されたもの) のみ、`examples/` または `fixtures/` 配下にコミットできます

## エクスポート機能で生成される JSON ファイル

`apps/local-web` の JSON エクスポート機能で生成されるファイル
(`profile-*.json`) は、`CareerProfile` のすべての項目を含みます。具体的には
氏名、フリガナ、生年月日、住所、電話番号、メールアドレス、職務経歴、学歴、
スキル、資格、プロジェクトなど、本ドキュメントが「コミット禁止」としている
個人情報そのものを含んでいます。

このため、エクスポートされた JSON ファイルの取り扱いは次のとおりとします。

- リポジトリにコミットしません (実データを含む `*.json` は `.gitignore` の
  `*.local.json` / `private/` / `data.local/` などで除外し、コミット対象から
  外してください)
- 第三者と共有する際は、内容が個人情報を含むことを送信者・受信者の双方で
  認識した上で扱ってください
- クラウド同期サービス、チャット添付、メールなど、コピーが残る経路で扱う
  場合は、保管期間と削除手順を事前に決めてください

エクスポートはユーザーの明示的な操作 (ボタンクリック) によってのみ発生し、
外部送信は行いません。生成された JSON はブラウザの download API でローカル
ファイルとして書き出されます。

## 画像データ (証明写真等)

- `CareerProfile.basics.profilePhoto.source.dataUri` に実画像データを含む `CareerProfile` JSON は、実履歴書データと同じ扱いとし、git にコミットしません (`.gitignore` の `*.local.json` / `private/` / `data.local/` で除外)
- サンプル / fixtures / docs に含める `dataUri` は、placeholder 画像 / 仮想バイト列 / 1×1 ピクセル等のダミーに限定します。実在人物の写真は使用しません
- `relativePath` の例も実在パスを書きません (例: `photos/profile.jpg` のような仮想例のみ)。絶対パス (`/Users/...` 等) は core が reject しますが、ドキュメント中にも書かないでください

## AI 連携の規則

`docs/CONCEPT.md` の原則に対応する、運用レベルの規則です。

- 外部 AI API はデフォルトで無効です。明示的に opt-in した場合のみ有効になります
- 任意の送信操作の前に、送信先 (ローカルモデル vs 外部 API) と送信内容をユーザーに提示します
- 既定では、ユーザーが選択した範囲のみを送信します (全文ではありません)
- 全文送信は明示的な確認ステップを必須とします
- AI とのやり取りのログ・プロンプト・レスポンスはローカルに留め、git 管理しません

## コミット前のチェックリスト

コミット直前に、以下を確認してください。

- [ ] 実在人物の氏名・連絡先・属性情報が含まれていないか
- [ ] 実在企業・組織の固有名が含まれていないか
- [ ] 秘密情報 (API キー、トークン、パスワード) が含まれていないか
- [ ] AI プロンプトやログをコミット対象に含めていないか
- [ ] 生成された PDF やバイナリを意図せず追加していないか
- [ ] `.gitignore` で除外すべき新規パスが追加された場合、`.gitignore` も更新したか

## 違反を発見した場合

すでに `main` にコミット・プッシュされた個人情報を発見した場合は、即座に問題を共有してください。Git 履歴からの除去はファイル削除では完了しません。`git filter-repo` 等による履歴の書き換え、認証情報のローテーションなど、別途対応が必要です。
